# 빅데이터 저장 및 관리기술
## S3
아마존 S#는 인터넷 스토리지 서비스, 웹에서 용량에 관계없이 데이터를 저장하고 검색할 수 있는 웹 서비스 인터페이스 제공
최소 1바이트에서 최대 5TB 데이터 객에의 읽기, 쓰기, 삭제가 가능하며, 저장 가능한 객체 수 제한이 없다. 아마존 S3는 데이터 보호 및 대용량
데이터 전송 기능을 지원한다. REST/SOAP 인터페이스를 사용하며 프로토콜과 기능 계층을 더욱 쉽게 추가할 수 있도록 유연하게 구성. 기본 다운로드 
프로토콜은 HTTP이며, BitTorrent 프로토콜 인터페이스를 추가로 제공 한다.
## HDFS(Haddop Distributed File System)
* 마스터노드는 네임노드라 불리며 데이터노드로 불리는 슬레이브 노드의 동작 상태를 실시간으로 관리하고 최대 수천대의 데이터노드에 분산되어 저장되어 있는 메타데이터를 관리함
* 데이터는 여러개의 블록으로 쪼개져 몇 대의 데이터노드가 한 블록을 복제해서 보관하는 방식으로 분산 저장
* 네임노드가 관리하는 메타데이터는 해당 블록이 어느 데이터노드에 저장되어 있는지의 여부 등을 파악

## DynamoDB
DynamoDB의 모든 데이터는 SSD에 저장되며, 하둡 기반 데이터 분석 기능과 데이터 저장 서비스 지원. DynamoDB의 기본 데이터 모델은 속성,항목,테이블로 구성된다.
속성은 <이름, 값> 쌍으로 구성된다. 이름은 문자열, 값은 문자열, 숫자 ,바이너리 등이 올수있다.
## MongoDB
* C++로 작성된 오픈소스
* 문서지향 (Document-Oriented) DB이며, 확장성과 고성능
* No SQL
* 데이터 구조는 한 개 이상의 key-value pair로 이루어져 있음
* Collection: MongoDB Document의 그룹
* Document들이 Collection 내부에 위치하고 있음
* 몽고 DB는 빅데이터 Iot 환경에 적합한 오픈 소스 플랫폼 모바일 응용 분야에 활용할 수 있는 다큐먼트 기반의 DB
* 몽고DB는 NoSQL 데이터베이스, JSON 형태의 데이터를 저장
* 몽고DB에서 저장하는 데이터 단위는 다큐먼트이며, RDBMS에서 행 단위의 레코드

### CRUD
* 몽고 DB 데이터 연산은 쿼리를 통하여 다큐먼트 데이터를 읽고 Create,Update, Delete 연산 동작을 사용한다 이를 CRUD라고 한다.
* 다큐먼트를 `JSON` 형태로 전달하면서 2진 JSON인 BSON 코드로 저장한다. BSON은 다른 다큐먼트 내부에 다큐먼트나 배열을 포함하는 것을 허용한다.
* 필드와 값을 조합해서 다큐먼트 형식으로 데이터를 저장한다. 
### 샤딩
여러장치 사이에 데이터를 저장하는 동작
* 수평분할 하여 물리적으로 서로 다른 곳에 분산 저장 및 조회하는것
* 몽고 DB에서는 대규모 출력 동작을 지원하기 위해 샤딩을 활용한다
## Cassandra
아마존의 DynamoDB의 해시 알고리즘을 이용한 키,값 저장 구조와 계층적 열구조를 논리 구조의 배경으로 한다
* 다른서버에 데이터 복제본을 구성하여 특정 노드에 장애가 발생해도 서비스에 영향을 주지않고, 데이터가 유실되지 않는다
* 수정,추가,삭제할 때 실제 스토리지 구조에 적용하기 전에 먼저 CommitLog에 변경 사항을 기록하므로 MySQL대비 8~15배 빠른 성능을 보인다.
* 1차 인덱스는 열 집합의 열 이름, 2차 인덱스는 열의 값을 기반으로 한다.
## Hbase
Hadoop의 HDFS위에 만들어진 분산 컬럼 기반의 데이터베이스, 구조화된 대용량의 데이터에 빠른 임의접근을 제공하는 구글의 빅 테이블과 비슷한 데이터 모델을 가지며
HDFS의 데이터에 대한 실시간 임의 읽기/쓰기 기능을 제공. 사용자는 HBase나 HDFS에 직접 데이터를 저장 할 수 있고, 사용자는 데이터를 읽고 접근하는 것은 HBase를
통한 임의접근을 이용
