## PIG 
야후에서 개발됐으나 현재는 아파치 프로젝트에 속한 프로젝트로서 복잡한 맵리듀스 프로그래밍을 대체할 피그 라틴 이라는 자체 언어제공
맵리듀스 API를 매우 단순화하 형태이고 SQL과 유사한 형태로 설계됨, 기존 SQL 지식을 활용하기가 어려운 편

## Hive
하둡에서 동작하는 SQL 프로그램을 구현할 수 있는 Hive는 하둡 기반의 데이터웨어하우스 인프라로, 관계형 데이터베이스에 익숙한 개발자에게
훌륭한 인터페이스를 제공함

## Cascading
* 하둡용 오픈소스 소프트웨어 추상화 계층
* 사용자들이 JVM 기반의 언어를 활용하여 하둡 클러스터에서 데이터 프로세싱 워크플로우를 제작하고, 제작한 프로그램을 실행 할 수 있도록 지원
* 광고 타깃팅이나 로그 파일 분석, 생물정보학, 기기 학습, 예측적 관점의 분석, 웹 콘테츠 마이닝, ETL 응용 프로그램 등에 주로 사용

## Cascalog
* `Clojure` 언어로 개발되어 기능적인 데이터 처리 인터페이스를 제공
* Datalog 언어의 영향을 받았으며, Cascading 프레임워크를 바탕으로 개발
* 하둡 클러스터에서 작은 규모의 데이터를 처리하는 코드 작성에 적합

## Mrjob
* 하둡 이후로 `Python`을 이용하여 하둡을 쉽게 프로그래밍할 수 있게 도와주는 프레임워크
* 미국의 맛집 추천 사이트로 유명한 Yelp에서 만든 Python 라이브러리
* 데이터를 처리하는 코드를 작성한 후 한곳에서 작동하는 엘라스틱 맵리듀스나 하둡 클러스터에서 동작하는 프레임워크를 제공

## S4(Simple Scalable Streaming System)
* 야후에서 공개한 오픈 소스 기반의 MapReduce 모델의 영향을 받은 분산 스트림 처리 시스템으로 야후의 서비스 운영에 활용
* 실시간 개인화 검색 광고의 선택과 위치 지정, 야후의 검색 서비스 제공 등에 필요한 데이터 마이닝, 기계학습 등을 지원하며, 임의의 이벤트 스트림 처리에도 응용할 수 있는 오픈소스
* ZooKeeper 프레임워클 사용하여 클러스터들의 분산 실행을 관리

## MapR
MapR은 빅데이터 플랫폼중의 하나로, Apache Hadoop을 사용하기 위한 통합 데이터 플랫폼이다. 간단하게 하둡등의 빅데이터 프로그램들을 편리하게 사용하기 위한 통합 플랫폼 이다.
아파치 하둡의 배포판 공급 업체인 MapR Technologies에서 높은 신뢰성을 요구하는 기업에 제공하는 상업용 하둡. HDFS를 대체할 수 있는 자체 파일 시스템으로 운영하며,
분산된 네임노드가 있어 개선된 신뢰성을 제공한다.

## Acunu
기존 파일시스템을 대체하여 MapR과 같이 새로운 저 수준의 데이터 저장 계층을 제공하는 하둡 보다는 Cassandra를 대체하려고 개발.
처리 속도를 개선하려고 빅데이터의 분석 질의, 범위 질의에 효과적인 계층적 (stratified) B-Tree를 구현하여 만든 저장 플랫폼.

## Azkaban
Linkedin의 오픈 소스 프로젝트로서 워크플로우 스케줄러, 시각화된 절차, 인증 및 권한 관리, 작업 모니터링 및 알람 기능

## Greenplum
Shared-Nothing MPP 구조의 데이터베이스이며, PostgreSQL을 기반으로 함. 표준 SQL 인터페이스를 활용할 수 있고, 다수의 시스템에 데이터를 복제하는
RAID 기술을 활용하여 데이터 손실도 최소화 함.

## EC2(Elastic Compute Cloud)
아마존 EC2는 클라우드에서 컴퓨팅의 규모를 자유롭게 변경할 수 있는 웹 서비스이며, 개발자가 웹 스케일 컴퓨팅을 쉽게 사용할 수 있도록 설계.
EC2는 웹 서비스 인터페이스를 사용하여 다양한 운영체제로 인스턴스를 시작. EC2는 메모리 및 CPU자원을 시간 단위로 지정하여 컴퓨터를 임대할 수 있는
웹 서비스. EC2는 슈퍼유저 권한으로 리눅스나 윈도우 서버 네트워크에 접근할 수 있으며, 소프트웨어 설치 및 시스템 설정이 자유로움.

## Pipes
Pipe는 야후가 제공하는 웹 응용프로그램으로 데이터 파이프라인을 생성
