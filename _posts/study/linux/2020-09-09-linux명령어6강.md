## 프로세스 관리
PID: 프로세스가 시작할 때 할당받는 프로세스 식별번호
Deamon: 시스템을 위해 또는 서비스를 위해 백그라운드에서 동작하는 프로세스
>프로세스 정보(/proc/PID/*)
{:.lead}

`PID`, `PPID`, `Contorl Terminal(TTY)`등 
**프로세스 관리1**
* 프로세스 실행
    - foreGround `# ls`
    - backGround `# ls &`

`백그라운드`로 프로세스를 실행하는 경우
1. 시간이 오래 걸리는 프로그램 실행(e.g. /root/bin/backup.sh &)
1. GUI 환경을 사용하는 프로그램
{:.note}

* 프로세스 확인
    ```bash
    # pa aux | grep rsyslogd
    # ps -ef | grep rsyslogd
    ```
* 프로세스 종료
    ```bash
    # kill -1 PID   (Daemon의 PID번호, 재기동)
        httpd
        httpd.conf(수정)
    # kill -2 PID   (# kill -INT PID, 인터럽트 시그널, Ctrl+C)
    # kill -9 PID   (# kill -KILL PID, 강제종료)
    # kill PID (# kill -15 PID(정상종료)
    ```
    killall CMD, pkill CMD
    {:.note}
**프로세스 관리2 (잡(Job) 관리)**
* 잡 실행
    ```bash
    fg) # ls
    bg) # ls &  
    ```
* 잡 확인
    ```bash
    # jobs
    # fg %1
    # bg %1
    CTRL + Z
    ```
* 잡 종료
    ```bash
    # kill %1
    ```
**프로세스 모니터링**
프로세스 모니터링에 top명령어를 사용하는데 해당 명령어의 해석이 중요하다.
```bash
# top
# top -u user01
```  
1. 정렬기능: c`P`u, `M`em(top 상태에서 해당 키워드 입력)
1. 첫번째 줄
    * uptime 출력내용 , `최소 300일`에 한번씩은 서버를 재부팅
    * 1분/5분/15분 부하측정치 (나누기) CPU 개수 >= 1
        = 0.05 (나누기) 4 >= 1
1. 두번째 줄
    * Task: 실행중인 프로세스
    * running: 실행중인 
    * sleeping: 대기중인 상태
    * stopped: 정지
    * zombie: 제대로 종료되지 않은 프로세스 
    * 좀비 프로세스 검색 방법
        ```bash
        # ps -elf | awk '$2 == "Z" {print $0}'
        ```        
1. 세번째 줄
    CPU상태를 나타냄
    * %us(usr), %sy(sys), `%id(idle): 유휴시간`, %wa(wait)
    * %us + %sy >= `80%`이면 CPU증설 고려     
1. 네번째 줄
    total / free 부분 중요
