## 빅데이터 컴퓨팅 개요
* 4차 산업혁명의 디지털 주요기술 : 빅데이터, 인공지능, IOT
	- 폭발적인 데이터 성장
    - 물건이나 장치에서 데이터를 생성
    - 서로 다른 데이터의 연관 관계 가능성
* 빅데이터로 부터 가치를 창출할 수 있는 가능성이 증가
	- 지속 가능한 빅데이터 혁신안을 내놓기 쉽지 않음
    - 신속한 행동을 하지 않으면 결과는 실패
* 빅데이터 효과: 인류가 직면한 의료,환경, 식량, 에너지 등의 분야에서 해결방안 제시

>빅데이터란?

**데이터베이스의 규모에 초점을 맞춘 정의(McKinsey)**<br>
일반적인 데이터베이스 sw가 저장, 관리, 분석할 수 있는 범위를 초과하는 규모의 데이터

**데이터베이스가 아닌 업무수행에 초점을 맞춘 정의**<br>
다양한 종류의 대규모 데이터로부터 저렴한 비용으로 가치를 추출하고 초고속 수집, 발굴, 분석을 지원하도록 고안된 차세대 기술 및 아키텍처

>빅데이터의 속성

빅데이터의 3V : `규모`, `속도`, `다양성`(정형,비정형,반정형)
4V : 정확성 5V: 가치

>빅데이터의 종류

    |종류|설명|
    |:---|:---|
    |정형|고정된 필드에 저장된 데이터(스프레드시트)|
    |반정형|고정된 필드에 저장되어 있지는 않지만, 메타데이터나 스키마 등을 포함하는 데이터(XML,HTML)|
    |비정형|고정된 필드에 저장되어 있지 않은 데이터 (이미지,동영상,음성|

## 출현배경
* 스마트폰 태블릿 등의 모바일 장치 확산
* 트위터, 페이스북 등의 소셜미디어 등장
* RFID와 같은 정보를 감지하는 센서 장비의 이용 확대
* Bioinformatics의 발전
* 메모리 비용 하락 
* 정보를 저장, 관리하는 `클라우드 컴퓨팅` 기술의 확산
* 하둡 파일시스템(HDFS) : 쉽고 싸게 사용할 수 있는 분산파일시스템

## 빅데이터 분석의 처리과정
>빅데이터 분석의 중요성

* 과학 기술 경쟁이 치열
* 첨단의 과학기술이 사용자의 요구를 충족시켰을 때 부가가치
* 미래 사용자의 필요 예측이 매우 중요
* 빅데이터 분석으로 미래 사용자의 필요 예측 가능

>. 데이터 소스: 내부 데이터, 외부데이터, 미디어
>수집: 수동, 자동

* 내부 데이터 수집 
   	내부 파일시스템, DB 센서
    ETL
* 외부 데이터 수집
   	인터넷으로 연결된 외부에서 데이터 수집
    crawling Engine
**수집 시스템의 요건**
1. 확장성
1. 안정성
1. 실시간성
1. 유연성
**빅데이터 수집 방법**
로그 수집기
크롤링
센싱
RSS Reader, Open API

>저장: 정형/ 비정형 

**저장 접근방식**
분산 파일시스템
NoSQL
병렬 DBMS
네트워크 구성 저장 시스템

>처리: 일괄처리, 실시간 & 배치

빅데이터에서 유용한 정보와 의미있는 지식을 찾아내기 위한 `데이터가공`이나 `데이터 분석과정`을 지원하는 과정

확장성,처리시간 단축 및 실시간 처리지원, 비정형 데이터 처리 지원

**빅데이터 처리기술**
* 빅데이터 일괄 처리 기술
* 빅데이터 실시간 처리 기술
* 빅데이터 처리 프로그래밍 지원 기술
>분석: 전처리, 분석방법

* 분석 시나리오 작성:  목적 정의, 시나리오 작성,절차 분석기법 등 세부 시나리오 작성
* 분석 환경 조사: 인프라 구축 조건등 분석
* HW 구축: 빅데이터 수용 용량 및 부하 등을 감안하여 HW 인프라 구축
* SW 구축: 분석에 필요한 수집, 관리, 분석, 이용자 환경 등 관련 SW구축

* 분석 실행

>표현(시각화)

구조화를 통해 `간단한 도표`나 `3D 이미지` 등으로 표현

## 하둡
하둡은 대량의 자료를 처리할 수 있는 대규모 컴퓨터 클러스터에서 동작하고, 분산 애플리케이션을 지원하는 오픈 자바 소프트웨어 프레임워크


- 하둡은 마스터하나와 슬레이브 여러 개로  클러스터를 구성
- HDFS에서는 마스터 노드를 `NameNode`라고 하며, 슬레이브 노드를 `DataNode`라고 함
- HDFS는 대용량 파일을 여러 `블록`으로 분산하여 저장, 기본 크기는 64MB
- HDFS는 데이터 복제 기법을 지원
