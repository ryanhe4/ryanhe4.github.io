## 빅데이터 컴퓨팅 기술
![빅데이터 플랫폼](http://postfiles8.naver.net/20160915_39/lool2389_1473903760003KQNz5_JPEG/%C4%B8%C3%B3.JPG?type=w773)

* 하둡(Apache Hadoop): 빅데이터 분산시스템, 특히 클라우드 컴퓨팅을 활용, 처리하기 위한 플랫폼
* 하둡의 특징
    - 분산 시스템의 확장성 구현: 하둡에서는 데이터의 위치를 추적하는 방식으로 분산 파일 시스템을 구현
    - 분산 시스템의 신뢰성 계승: 일부 서버가 고장 나도 중단 없이 작동이 가능
    - 오픈소스로 제작되어 시스템 구축에 큰 비용이 들지 않음
* 하둡의 장점
    - 대용량 데이터 저장
    - 데이터 무결성
    - 장애 복구
    - 스트리밍 방식의 데이터 접근    
* 빅데이터 애널리틱스: 빅데이터로부터 통찰을 얻기 위해 빅데이터를 분석, 표현할 수 있는 기술
    - 빅데이터 분석기술: 데이터마이닝, 기계학습, 텍스트마이닝, 오피니언마이닝, 소셜네트워크 분석 등
    - 빅데이터 표현기술: 동적 데이터 시각화
    - 관련 소프트웨어: SAS, SAP, 오라클, HP, IBM등 주요 소프트웨어 기업은 하둡을 기반으로 데이터를 분석할 수 있는 도구 제공

### R프로그램
* R 프로그램: 최신의 분석 알고리즘을 유지, 인메모리 계산으로 데이터를 빠르게 처리할 수 있음
    - 하둡 환경의 맵리듀스 기능을 R프로그램 코드로 구현 -> R을 이용하여 하둡 기반의 분석을 할 수 있음
* R은 인메모리 계산을 하기 때문에 큰 용량의 데이터를 처리하는 데 제약
    - 빅메모리 패키지와 빅애널리틱스 패키지를 이용하여 해결
    - 두 패키지 모두 데이터의 구조만 물리적 메모리에 저장하여 마치 데이터가 물리적 메모리에 로드되어 있는 것처럼 처리

## 빅데이터 수집
* 빅데이터 분석을 위해서 내부 또는 외부의 다양한 원천으로부터 자동 또는 수동으로 데이터를 수집
* 외부데이터는 정부 또는 데이터 기업에 의해 공개(공개 API 또는 파일)
* 데이터의 수집: 기업 및 정부의 내부 데이터와 외부의 다양한 원천 데이터를 검색하여 수집하고 이를 변환, 통합하는 것
* 내부 데이터 중 웹서버에 있는 로그 데이터는 오픈소스인 스크라이브(Scribe), 척와(chukwa), 플룸(flume) 등 로그 수집기에 의해 수집
* 외부 데이터는 외부 데이터베이스 또는 포털에서의 `검색`, `공개 API`를 이용한 검색, 인터넷에 공개되어 있는 웹페이지의 `크롤링`을 통해 수집 

### 검색 데이터 수집
* 네이버 트렌트, 구글 트렌드
* 네이버 트렌드를 이용하여 맛집, 식당 키워드 검색결과를 시간대별로 정리하여 사람들의 패턴 파악
* 구글 독감 트렌드: 집계된 구글 검색 데이터를 사용하여 전 세계 독감 유행 수준을 실시간으로 예측한 결과를 보여 줌
* 구글 Ngram : 구글 북스를 검색할 수 있는 프로그램

### 웹페이지 데이터 수집
웹페이지는 텍스트, 이미지, 동영상 등 다양한 정보를 보여주기 때문에 데이터 분석에 적합하지 않음
- 웹 스크래핑을 이용하여 HTML로 작성된 웹페이지에서 필요한 데이터만 추출
- 웹 크롤러
- HTML 파싱
### 트위터 데이터 수집
트위터는 대표적인 소셜네트워크서비스로 트윗(tweet)이라고 불리는 140자의 짧은 문자을 공유하는 마이크로블로그서비스
#### 트위터에서 API 이용
트위터는 외부에서 트위터의 기능을 이용하여 제3자 애플리케이션을 개발하거나 트위터 데이터를 수집할 수 있도록 공개 API를 제공

### 페이스북 데이터 수집
페이스북은 메세지, 사진, 동여상등 다양한 정보 공유
- 사용자의 친밀한 그룹을 중심으로 정보가 공유되는 특징
- 트위터에서의 빅데이터 연구가 정보의 확산 효과에 초점이 맞추어진 반면, 페이스북에서의 빅데이터 연구는 사용자의 소셜네트워크상에서의 행위에 더 초점을 맞춤
